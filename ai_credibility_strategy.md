## The Core Problem with AI Fitness Plans
Currently, when trainers use ChatGPT to write diet or workout plans, the output is generic, unreferenced, and often hallucinates macronutrient math. Clients can tell it was generated by AI, which damages the trainer's perceived value. If a client is paying 4,800 AED for a Grand Slam Offer, the deliverables must feel premium, bespoke, and scientifically grounded.

## The Solution: The "Two-Brain" Architecture
To solve this, TrainedBy Pro will not just use a standard LLM. We have designed a **Two-Brain Architecture** that separates *scientific research* from *plan formatting*.

### Brain 1: Perplexity Sonar Pro (The Researcher)
Before any plan is written, the system queries Perplexity's `sonar-pro` API. Sonar Pro is a search-grounded model that actively browses the internet in real-time.
*   **The Prompt:** We ask Sonar to find the latest peer-reviewed guidelines (from the NSCA, ACSM, or ISSN) for the client's specific demographic and goal (e.g., "ISSN protein recommendations for a 45-year-old female in a caloric deficit").
*   **The Output:** Sonar returns a dense, evidence-based summary and, crucially, **real citations** (URLs to PubMed, NIH, or sports science journals).

### Brain 2: OpenAI GPT-4.1 (The Formatter)
We then pass the client's profile, the calculated TDEE (Total Daily Energy Expenditure), and the research context from Sonar into OpenAI's `gpt-4.1-mini`.
*   **The Prompt:** We instruct GPT to act as an elite sports dietitian and format a structured JSON plan that strictly adheres to the science provided by Sonar.
*   **The Output:** A perfectly structured, culturally relevant (e.g., including UAE foods like labneh and dates), and mathematically accurate plan.

## How This Builds Credibility for the Trainer

When the trainer hands this plan to their client, it does not look like a ChatGPT output. It looks like a clinical document.

1.  **The "Science Note":** Every plan includes a summary of the methodology. *(e.g., "This hypertrophy block is designed using progressive overload principles aligned with the 2024 NSCA guidelines for volume load.")*
2.  **Real Citations:** The plan includes a "Research Sources" section at the bottom, listing the actual studies Sonar found. This is the ultimate credibility flex for a trainer.
3.  **Evidence-Rated Supplements:** Instead of just saying "take creatine," the plan grades supplements based on the Australian Institute of Sport (AIS) framework (Grade A = Strong Evidence, Grade B = Emerging, etc.).
4.  **Trainer Branding:** The entire document is branded with the trainer's name and the TrainedBy logo, making the trainer look like they have a team of sports scientists working for them.

## Technical Implementation Details

The prototype has been built in `/supabase/functions/generate-plan/index.ts` and `/plan-builder.html`.

### 1. The Edge Function Workflow
*   **Input:** The frontend sends the client profile (age, weight, goal, dietary restrictions, injuries).
*   **Math:** The edge function calculates the BMR (Mifflin-St Jeor equation) and TDEE based on activity level, then sets the caloric target based on the goal.
*   **API Call 1 (Perplexity):** Fetches the scientific context and citations.
*   **API Call 2 (OpenAI):** Generates the JSON structure using `response_format: { type: "json_object" }` to ensure the frontend never breaks.
*   **Output:** Returns the structured JSON to the frontend.

### 2. Cost Economics
*   **Perplexity Sonar Pro:** ~$3.00 per 1M input tokens. A typical query uses ~100 tokens.
*   **OpenAI GPT-4.1-mini:** ~$0.15 per 1M input tokens, $0.60 per 1M output tokens. A typical plan uses ~800 input and ~1,500 output tokens.
*   **Total Cost per Plan:** Less than **$0.005 (0.018 AED)**.
*   **ROI:** We include this tool in the TrainedBy Pro tier (199 AED/month). If a trainer generates 20 plans a month, our cost is 0.36 AED. The margin is effectively 100%.

How does a trainer set their availability?

This determines how consumers see open slots and how complex the trainer setup is

A

Weekly recurring slots

Trainer picks which days and hours they're available each week. Repeats automatically. A consumer sees "Tuesday 9am, Tuesday 10am, Thursday 2pm…"

Pros

  • 5-minute setup
  • No third-party auth
  • Simple to display

Cons

  • Trainer must manually block holidays
  • No sync with their real calendar

→ Recommended for v1

B

Google Calendar sync

Trainer connects Google Calendar via OAuth. Availability is derived from their free/busy. Bookings appear as events automatically.

Pros

  • Zero double-booking risk
  • Real-time accuracy
  • Familiar to trainers

Cons

  • OAuth integration (2-3 weeks extra)
  • Google API quota management
  • Trainer must have Google account
C

Weekly slots + manual overrides

Recurring weekly schedule as a base, but trainer can block specific dates (vacations, sick days) and add one-off slots without changing the pattern.

Pros

  • Flexible without full calendar sync
  • Handles holidays cleanly
  • Still no OAuth required

Cons

  • Slightly more complex UI
  • ~1 extra week to build vs A

→ Best long-term without Google dependency

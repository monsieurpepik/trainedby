Booking System — Architecture Overview

Three actors, one clean payment flow

The flow

Trainer (Pro plan)
  • Connects Stripe account (one-time)
  • Creates session types + prices
  • Sets weekly availability
  • Blocks dates when needed
  • Sees bookings in dashboard
Consumer
  • Finds trainer on TrainedBy
  • Clicks "Book a Session"
  • Picks session type + time slot
  • Leaves name, email, phone, goal
  • Pays via Stripe
  • Gets confirmation email
Platform (TrainedBy US)
  • Stripe Atlas entity processes payments
  • Zero commission — trainer keeps 100%
  • Stripe routes money to trainer directly
  • Sends confirmation + reminder emails
  • Handles refunds if cancelled 24h+

Session types supported at launch

Single session
Pay once, book one slot. e.g. "60-min PT session — $80"
Package
Pay upfront for N sessions at a discount. Credits stored, scheduled one at a time. e.g. "10-session pack — $700"
Memberships (recurring billing) — planned for v2, infrastructure compatible
Cancellation policy (platform-wide): Full refund if cancelled 24h+ before session. No refund if cancelled within 24h. Enforced automatically via Stripe refund API.
